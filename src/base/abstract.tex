We present a system that recommends which transport implementation to deploy
for a given network, choosing among four QUIC implementations and a TCP
baseline. No implementation is fastest everywhere, and QUIC sometimes loses to
TCP, so the best choice depends on the network conditions and on the
deployment's priorities. We build a Docker and \texttt{tc/netem} benchmarking
framework, measure all five across a wide grid of conditions into a
216,000-run dataset, and train a two-stage random forest: the first stage
discards implementations likely to time out, the second ranks the rest for a
chosen profile. A live prober applies the trained model to a freshly measured
connection. Our evaluation shows that the best choice shifts with conditions
and priorities, that the model predicts it accurately, and that the prober
works in practice.
